## 任务 1：提取目录（TOC）→ JSON

### 目标

每个目录生成 `output/{book}/chapter-map.json`，格式：

```json
[
  {"title": "第1章 有理数", "level": 1, "page": 1},
  {"title": "1.1 从自然数到有理数", "level": 2, "page": 1},
  {"title": "知识点 1 自然数的应用", "level": 3, "page": 3}
]
```

### 执行方式

**8 个 subagent 并行**，每个处理一个目录：

1. 读取 `output/{book}/raw_pages/{主书PDF}/doc_0.md` ~ `doc_4.md`（前 5 页）
2. 找到目录页（通常 doc_3.md 或 doc_4.md，有 `## 目录` 标记）
3. 用 subagent-prompt.md 的**封面 & 目录解析模板**提取
4. 保存到 `output/{book}/chapter-map.json`

### 已知风险

- 七年级上册 B本的目录页有 OCR 幻觉（"几何图形性质"重复 121 次）— subagent 需要去重
- 部分小学教材目录用 `#####` 格式而非 `第N章` 格式 — subagent 需自适应

## 任务 2：每目录上传 200 道题

### 目标

每个目录的「题库」表写入 200 条记录，含：
- 题干、题型、选项、答案、解析
- 知识点标签、思想标签、模型标签（从内容推断）
- 所属章节（关联目录表）
- 题目图片（附件）

### 答案匹配策略

**核心逻辑**：先解析答案 PDF 建立 lookup，再解析内容 PDF 时查表填答案。

#### Step A：解析答案 PDF

对每个目录下的答案 PDF（doc_0.md ~ doc_N.md）：
1. 跳过封面（doc_0）和目录（doc_1）
2. 从 doc_2 开始解析答案内容
3. 提取格式：`题号 → 答案 + 解析`
4. 保存到 `output/{book}/answers/{pdf_name}.json`

答案解析规则：
- `## 1. A` → 题号 1，答案 A
- `2. 45` → 题号 2，答案 45
- `4. (1) D (2) B` → 题号 4，子题 (1)=D, (2)=B
- `解析` 后的文字 → 解析字段
- 选择题答案：A/B/C/D
- 填空题/解答题：保留数值表达式

#### Step B：解析内容 PDF + 匹配答案

对每个目录的内容 PDF（doc_5.md 起）：
1. 用 subagent-prompt.md 的**题目解析模板**解析
2. 滑动窗口：每次传 3 页，解析中间页
3. 每页平均 5-8 道题，取前 **25-40 页** = 200 道题
4. 从答案 lookup 中匹配：`section_title` + `题号` → 答案

#### Step C：写入飞书 Base

对每批 ≤200 道题：
1. `+record-batch-create` 写入题干、题型、选项、答案、解析、标签等字段
2. 获取 `record_id_list`
3. 对有图片的题目，`+record-upload-attachment --field-id "题目图片"` 上传

### 执行方式

**8 个目录串行处理**，每个目录内：
1. **1 个 subagent** 解析所有答案 PDF → answers.json
2. **3-5 个 subagent 并行** 解析内容 PDF（每个处理 10-15 页）
3. **主流程** 写入飞书 Base（batch-create + upload-attachment）

### 字段写入映射

| 字段 | 来源 | 格式 |
|------|------|------|
| ID | 自动编号 | 不写 |
| 题干 | subagent 解析 | 字符串，LaTeX 用 `$...$` |
| 题型 | subagent 推断 | 选择题/填空题/解答题/计算题/应用题 |
| 选项 | subagent 解析 | `"A. xx\nB. xx\nC. xx\nD. xx"` |
| 答案 | 答案 PDF 匹配 | 字符串（选择题字母，填空题数值） |

| 难度 | subagent 推断 | 1-5 |
| 有确定解 | subagent 判断 | true/false |
| 知识点 | section_title 推断 | 字符串数组 |
| 知识点标签 | section_title 推断 | 字符串数组（预置选项） |
| 思想标签 | 内容推断 | 字符串数组（预置选项） |
| 模型标签 | 内容推断 | 字符串数组（预置选项） |
| 所属章节 | 章节匹配 | `[{"id": "rec_chapter_xxx"}]` |
| 年级 | base-info.json | 字符串 |

| 来源 | PDF 文件名 | 字符串 |

## 执行顺序

1. **Phase 1（并行）**：8 个 subagent 同时提取 8 个目录的 TOC → chapter-map.json
2. **Phase 2（并行）**：8 个 subagent 同时解析 8 个目录的答案 PDF → answers.json
3. **Phase 3（串行 × 8）**：每个目录：
   a. 3-5 个 subagent 并行解析内容页（取 200 道题）
   b. 主流程 batch-create 写入飞书
   c. 主流程 upload-attachment 上传图片
4. **Phase 4**：验证 — 每个目录随机抽 5 条记录检查质量

## 验证标准

- 每个目录 ≥200 条记录
- 选择题答案正确率 ≥80%（有确定解的题目）
- 图片成功上传（题目图片字段非空）
- 所属章节关联生效（飞书 UI 可点击跳转）
- 标签字段有值（至少填了知识点标签）

## 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| TOC 页 OCR 幻觉严重 | 目录提取不准 | subagent 去重 + 人工抽检 |
| 答案 PDF 格式不一致 | 答案匹配失败 | 先解析 2 个答案 PDF 验证格式 |
| 题目跨页截断 | 题干不完整 | 3 页滑动窗口 |
| 附件上传幂等性问题 | 图片重复 | 每记录只上传 1 次 |
| lark-cli 绝对路径问题 | 命令失败 | 全部用相对路径 `@./xxx.json` |
| subagent 超时/失败 | 部分页面未解析 | 重试 + 跳过已成功的 |
