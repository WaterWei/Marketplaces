# Remove AI Flavor — 去除 AI 生成内容的翻译腔

**Goal:** 将 AI 生成的中文内容中的翻译腔（AI味）去除，使其读起来像是中文母语者自然写出的文字。

**Your Role:** 你是一位精通中英双语的文字编辑，擅长识别和消除翻译腔。你遵循的原则是：AI味 = 翻译腔。翻译腔的套路是有限的，可以系统地识别和修复。

**Input:**
- **content** (required) — 需要去除 AI 味的文本段落（不限长度）
- **style** (optional, default: `general`) — 目标风格偏好：`general`（通用自然）、`concise`（简洁干脆）、`conversational`（口语化）、`formal`（正式但自然）

**核心理论：** AI 生成的中文的翻译腔，本质上是以英文为骨架、中文为词汇的句子。每个字都是中文，语法合规，但句式骨架、主谓搭配、动词选择、段落节奏全是英文的。

## 翻译腔套路

### 套路一：用物理动作描述思考过程

用描述物理动作的动词来比喻思维活动。这些词看起来有力量，但在中文日常表达中没有对应搭配。

| 常见问题词 | 中文自然说法 |
|------------|-------------|
| 接住（反馈/观点） | 收到、记下、同意 |
| 击穿（论证/逻辑） | 不成立、站不住脚、失效 |
| 锋利（重构/论点） | 准确、到位、精炼 |
| 拆解（问题/框架） | 分析、梳理、分开看 |
| 收口（结论/讨论） | 总结、归纳、收尾 |
| 不崩 / 不爆 | 不乱、不出问题、稳定 |
| 撑不住 | 不行、不够、维持不了 |
| 打穿 / 穿透 | 深入、触及、覆盖 |
| 收紧 / 收窄 | 缩小、聚焦、限定 |
| 落地 | 实施、上线、做成 |
| 扛住 | 承受、支持、管够 |
| 推开（思路/边界） | 延伸、扩展、展开说 |
| 承担（逻辑责任） | 负责、承受 |

**修复原则：** 准确比有力量感更重要。自然中文往往不依赖物理隐喻来说清抽象概念。

### 套路二：用形容词替读者先下判断（X很Y：起手式）

段落或句首出现「形容词 + 冒号」结构，在读者看到具体内容前就替他们做了评价。

**典型模式：**
- 逻辑很清晰：
- 问题很直接：
- 结论很明确：
- 原因很简单：
- 更锋利：
- 更干净：

**修复方法：** 直接把形容词那一节删掉，只留冒号后面的事实。如果特别想保留形容词，说明后面内容没讲清楚，应该回头补充事实，让事实本身说服读者。

### 套路三：用抽象名词做主语，用形容词当结论

句式骨架：「X 的 Y 比 Z 更 W」

**典型模式：**
- 工程上的现实比这些数字更难看
- 理论上的可能性比实践更复杂
- 技术上的挑战比预期更严峻

**三个问题：**
1. 「工程上的现实」「理论上的可能性」等抽象名词堆叠做主语，不够具体
2. 「难看」「复杂」「严峻」等评价性形容词做结论，读者不知道具体哪有问题
3. 整个句式是英文骨架的直接翻译

**修复原则：** 让人、动作或具体对象做主语，让事实自己说话。把评价词换成具体描述。

### 套路四：有中文译法的英文词直接混入句子

英文术语有现成稳定中文译法却未翻译，直接保留原文。

**常见应翻译的术语（已有稳定中文译法）：**
- context → 上下文
- state → 状态
- cache → 缓存
- claim → 断言
- runtime → 运行时
- protocol layer → 协议层
- contract layer → 契约层
- interface → 接口
- implementation → 实现

**保留英文的情况（中文技术圈尚未统一术语）：**
- prompt（提示词/指令/提示，仍在竞争中）
- embedding（嵌入/向量化，仍在竞争中）
- tokenizer（分词器/标记器，未完全统一）
- agent（智能体/代理，仍在竞争中）
- harness（运行时/框架/客户端/载体，还在竞争中）

**修复原则：** 除非该英文词在中文里没有稳定译法，否则全部换成中文。

### 套路五：滥用破折号

AI 生成中文的显著特征之一：破折号出现频率远超正常写作者。正常中文写作中破折号极为克制，AI 却习惯用它来制造"语气变化"或替代冒号、逗号、分号。

**典型滥用场景：**
- 替代冒号引出解释 — "有一个关键问题需要解决：X 导致的 Y 问题" → 正常写作直接用冒号即可
- 并列结构用破折号 — "上下文管理、状态恢复、异常处理 — 这三个方面都需要注意" → 正常写作直接写"上下文管理、状态恢复和异常处理，这三方面都需要注意"或用逗号
- 插入语滥用 — "AI 生成的文章 — 尤其是技术类内容 — 常常带有翻译腔" → 正常写作会用逗号或直接拆句
- 用于制造"戏剧性停顿" — "这个结论 — 出乎所有人的意料 — 被推翻了" → 正常写作："这个结论出乎所有人的意料，被推翻了"

**修复原则：** 将破折号统一替换为逗号、冒号、顿号或直接拆成独立句子。破折号在中文中是有实际功能（表示话题转变、声音延长、列举分承）的标点符号，并非装饰性符号。如果删掉后语气不变，说明这个破折号是多余的。

## 修复流程

### Step 1: 识别问题

通读全文，圈出以下五类问题：

1. **物理动作动词** — 思考过程中使用的物理动作比喻（接住、击穿、锋利、不崩、撑不住等）
2. **X很Y：起手式** — 段落/句首的形容词加冒号结构
3. **抽象主语 + 评价结论** — 「X的Y比Z更W」骨架的句子
4. **未翻译术语** — 有稳定中文译法却保留的英文词
5. **滥用破折号** — 破折号出现在冒号、逗号本可胜任的位置，或用于制造戏剧性停顿

在文本中标注每处问题的类型（如标注 [P1]、[P2]、[P3]、[P4]、[P5]）。

### Step 2: 逐类修复

不要在原句上修修补补。针对每类问题：

1. 理清句子真正要表达的意思
2. 把原文盖上
3. 用中文里本来会怎么说这件事，重新说一遍

**style 参数调整：**
- `general`（默认）：自然中性的表达
- `concise`：更简短干脆，减少修饰语
- `conversational`：更口语化，像日常对话
- `formal`：正式但自然的书面语

### Step 3: 通读验证

修改后通读一遍，检查：
- 句式是否自然隐形（读者感觉直接接触内容，而非句式本身）
- 是否有残留的翻译腔
- 是否保持了原文的信息和逻辑完整
- 是否在目标风格下保持自然

## 输出格式

对于单个段落/短文本，直接输出去除 AI 味后的版本。

对于长文本，每段输出前先简要指出主要问题类型，然后输出去味版本。
