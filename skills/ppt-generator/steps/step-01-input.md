# Step 1: 收集输入

## 目标

获取用户的演讲稿/文稿内容，确认输入格式和基本元信息。

## 执行流程

### 1. 加载 Schema

**必须先读取** `{skill-root}/schema.md`，将完整的 JSON Schema 作为后续所有步骤的工作基准。

### 2. 判断输入来源

检查用户是否已经提供了内容：

**A) 文件路径** — 用户指向了一个 `.md` / `.txt` / `.doc` 文件：
- 读取文件全部内容
- 确认内容非空

**B) 直接文本** — 用户在对话中粘贴了演讲稿内容：
- 直接使用该内容

**C) 未提供内容** — 用户只说了"帮我生成 PPT"但没给内容：
- HALT，要求用户提供：
  1. 演讲稿文件路径，或
  2. 直接粘贴演讲稿文本，或
  3. 描述演讲主题和大纲，由 AI 辅助生成

### 3. 收集元信息

向用户确认以下信息（如已从内容中推断出，列出并请用户确认）：

1. **演示标题** — 从内容推断或询问
2. **作者** — 可选，询问是否需要
3. **日期** — 可选，默认当前年份
4. **目标时长** — 约 30 分钟 / 60 分钟 / 90 分钟（影响幻灯片数量）
5. **目标受众** — 影响语言风格和深度

### 4. 确认大纲

基于输入内容，快速列出识别到的**主要章节**（3-8 个），以编号列表呈现。这一步的目的是让用户确认结构拆分是否合理，不做详细布局设计。

示例输出：
```
我识别到以下章节结构：
1. 封面
2. 背景与痛点
3. 核心概念介绍
4. 技术架构
5. 代码示例
6. 与现有方案对比
7. 落地计划
8. 总结与讨论
```

HALT，等待用户确认或调整大纲。
