# Step 2: 内容分析与布局规划

## 目标

将确认的大纲映射为具体的幻灯片序列，为每张幻灯片选择最佳布局和内容结构。

## 执行流程

### 1. 重读 Schema

回顾 `{skill-root}/schema.md` 中的**布局选择指南**，确保布局决策有据可依。

### 2. 逐章节分析

对用户确认的每个章节，执行以下分析：

**a) 识别内容特征：**
- 是否有 3-6 个并列要点？ → `cards`
- 是否是有序步骤/操作清单？ → `list`
- 是否有 2-3 个维度需要并列展示？ → `columns`
- 是否包含代码片段？ → `code`
- 是否是结构化数据/矩阵？ → `table`
- 是否是正反/方案对比？ → `comparison`
- 是开场或结尾？ → `cover`
- 以上都不是？ → `content`

**b) 判断是否需要拆分：**
- 一个章节内容过多（超过 7 个要点）→ 拆分为 2 张幻灯片
- 一个章节既有概述又有细节 → 拆分为概述页 + 细节页
- 一个章节包含多种内容类型（如文字 + 代码）→ 拆分为多张

**c) 确定 eyebrow 标签：**
- 使用简短英文标签概括该幻灯片的角色
- 常用标签：`OVERVIEW`、`INTRO`、`ARCHITECTURE`、`CODE`、`COMPARISON`、`ROADMAP`、`Q&A`、`SUMMARY`、`DEMO`、`PHASE 1` 等

### 3. 输出布局规划表

以表格形式呈现完整的幻灯片规划，格式如下：

```
| # | eyebrow | 标题 | layout | 内容摘要 |
|---|---------|------|--------|---------|
| 1 | (空)    | 演示标题 | cover | 封面：标题 + 副标题 + 日期 |
| 2 | WHY     | 痛点分析 | cards | 3 个痛点卡片 |
| 3 | OVERVIEW | 概述 | content | 核心概念描述 + 3 个要点 |
| ... | | | | |
| N | Q&A    | 讨论 | content | 讨论引导问题 |
| N+1 | (空)  | 谢谢 | cover | 结尾页 |
```

### 4. 检查幻灯片数量

根据 Step 1 确认的目标时长，检查幻灯片数量是否合理：

| 时长 | 建议页数 | 节奏 |
|------|---------|------|
| 30 分钟 | 12-18 页 | 快节奏 |
| 60 分钟 | 25-35 页 | 标准节奏 |
| 90 分钟 | 35-50 页 | 深度节奏 |

如果页数不在合理范围内，向用户说明并建议调整。

HALT，等待用户确认布局规划表。用户可能会：
- **确认** → 进入 Step 3
- **调整**（合并/拆分/重排/换布局）→ 修改后再次确认
- **要求增减内容** → 重新分析受影响的章节
