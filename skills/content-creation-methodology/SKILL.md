# 内容创作方法论

**核心理念：做内容的本质是讲故事。** 信息搬运（转述）≠ 内容创作。内容创作是把"原材料"加工成"成品"，让读者看完后觉得世界跟之前不太一样了。

三步框架：**获取信息 → 找角度 → 创作**

**输入**：选题方向 / 热点事件 / 已有素材 / 用户模糊想法
**输出**：结构化的内容大纲 + 角度策略 + 节奏方案

---

## On Activation

当检测到触发词时，执行以下步骤：

### Step 0.1：定范围

向用户确认三个信息：
1. **内容载体**：公众号长文 / 短视频脚本 / 小红书图文 / 其他？
2. **主题或选题**：已有明确选题 / 需要我帮你找角度 / 只有一个模糊想法
3. **目标读者**：行业专业人士 / 普通大众 / 特定圈层？

### Step 0.2：确认方法论路径

告知用户将按"获取信息 → 找角度 → 创作"三步走，每一步需要用户参与关键决策。征得同意后开始。

---

## 第一步：获取信息

### 目标
在动笔前，建立足够多元的信息基础，避免信息偏科导致表达干瘪。

### 执行动作

1. **问用户已有信息**：关于这个选题，你已经掌握了哪些素材/信息？
2. **跨领域扫描**（引导用户思考）：
   - 这个选题在行业内的常规报道角度是什么？
   - 有没有历史/商业案例可以做类比？（读史可以鉴今）
   - 有没有电影、综艺、小说的叙事结构可以借用？（喜剧的升番、电影的情绪曲线）
   - 有没有日常生活观察可以切入？（人性藏在具体琐碎里）
3. **信息拼图**：把上述信息整合，标出"已知信息"和"信息盲区"

### 输出物
一段信息摘要，列出：
- 核心事实（行业资讯/热点事件）
- 跨领域关联素材（历史类比/故事框架/生活观察）
- 信息盲区（需要进一步了解的）

---

## 第二步：找角度（占 60% 权重）

### 目标
找到"情理之中，预料之外"的独家角度——让读者对一个已知信息产生全新的感知。

### 执行动作

1. **头脑风暴常规角度**：列出所有人都会想到的切入方向
2. **逐一否决**：这些角度全部不能写，因为所有人都能想到
3. **寻找反差**（引导用户）：
   - 大家都在说 A 面，B 面是什么？
   - 大家都在从上往下看，从下往上看是什么样？
   - 大家都在讲成功，失败/挣扎/代价呢？
   - 大家都在讲技术本身，人的故事呢？
   - 第一时间想到的角度 ×，第二时间想到的角度可能还不够，第三第四呢？
4. **验证角度**：这个角度是否符合"情理之中，预料之外"？读者看到会不会"卧槽，有道理"？

### 输出物
一个明确的角度陈述（一句话）：
> 本文将从「______」这个角度切入，让读者看到______。

---

## 第三步：创作

### 目标
设计内容的节奏和结构，确保有起有伏，同时守住底线。

### 执行动作

1. **设计节奏**
   - **定情绪曲线**：画出内容的波峰波谷——哪里让读者兴奋、哪里让读者沉思、哪里让读者意外
   - **用升番**：如果是三段式结构，每一段要比上一段更精彩。第一段让读者觉得"有点意思"，第二段"确实不错"，第三段"卧槽"
   - **开头定生死**：开头 3 句能不能抓住读者？有没有悬念/反差/故事钩子？

2. **框架推荐（选其一）**
   - **英雄之旅**：平凡世界 → 冒险召唤 → 跨越门槛 → 考验 → 高潮 → 回归
   - **问题-解决方案**：痛点描述 → 分析根因 → 解决方案 → 升华
   - **故事弧线**（Kurt Vonnegut）：横轴时间、纵轴"处境好坏"，确保有起伏

3. **价值观审核**（底线）
   - [ ] 这个选题涉及的知识我是否真的懂？——不懂的不要碰
   - [ ] 是否涉及敏感话题？——不要碰
   - [ ] 语气是否在"秀优越"而不是"分享"？——保持弱者思维，你只是爱分享

### 输出物
- 内容大纲（段落级结构 + 情绪曲线标注）
- 开头段落（试写）
- 节奏标注（哪里是升番点、哪里是情绪转折）

---

## 质量检查清单

交给用户前，逐项自检：

- [ ] **信息充足**：信息来源是否跨领域？有没有只依赖单一信息源？
- [ ] **角度独到**：这个角度是否避开了"所有人都会想得到"的方向？是否符合"情理之中，预料之外"？
- [ ] **节奏有起伏**：从头到尾读一遍，情绪是否有波动？还是像说明书一样平？
- [ ] **开头有钩子**：前三句能让读者想继续看吗？
- [ ] **价值观安全**：不懂的知识点碰了吗？敏感话题涉及了吗？语气在俯视读者吗？

---

## 对话策略

| 用户状态 | 应对方式 |
|---------|---------|
| 有明确选题 | 跳过 Step 1 信息收集，直接进入 Step 2 找角度 |
| 只有模糊想法 | 先用 Step 1 帮用户理清信息边界，再进入 Step 2 |
| 已经写了初稿 | 直接用质量检查清单跑一遍，给出修改建议 |
| 不知道写什么 | 帮用户找选题：最近有什么热点？你最近在做什么？有什么想表达的？ |

## 调性指南

- 不讲空泛的理论，每个观点必须有案例支撑
- 保持"分享者"姿态，不是"布道者"姿态
- 具体 > 抽象，故事 > 说理，细节 > 概括
