# Step 1: 读取输入 & 收集元数据

## 目的

读取用户提供的大型 Markdown 文件，收集 PPT 元数据，确认分割配置。

## 执行步骤

### 1. 加载 Schema

读取 `{skill-root}/schema.md`，了解完整的 JSON 格式定义。后续所有步骤都基于此 schema。

### 2. 确定输入文件

检查用户提供的参数：

- **A) 文件路径** — 用户提供了 `.md` 文件路径，读取完整内容。
- **B) 直接文本** — 用户粘贴了内容，将其保存为临时 `.md` 文件。
- **C) 无内容** — 用户只说了"文档转ppt"但没给文件。**HALT**，要求用户提供文件路径。

记录：文件路径、文件大小（字符数和行数）。

### 3. 收集元数据

向用户确认以下信息（有默认值的可以跳过）：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `title` | 从文档第一个 `#` 标题提取 | PPT 标题 |
| `author` | 空 | 作者（可选） |
| `date` | 当前年份 | 日期 |

### 4. 确认分割配置

读取 `{skill-root}/customize.toml` 中的 `split` 配置，向用户展示：

- 当前配置的 section_types（有哪些分段模式）
- max_concurrency（并发数）

提示用户：如果文档结构与默认配置不匹配，可以修改配置或提供自定义配置文件路径。

### 5. 估算分片数

快速扫描文档标题，估算能匹配到多少个分段。向用户报告：

```
预估分片数: XX 个
并发数: XX（max_concurrency）
预计批次数: XX
```

### CHECKPOINT

展示以上信息。**HALT** 等待用户确认。

用户确认后，记录元数据变量：
- `{input_file}` — 输入文件路径
- `{title}` — PPT 标题
- `{author}` — 作者
- `{date}` — 日期
- `{config_path}` — 配置文件路径
- `{output_dir}` — 输出目录路径
